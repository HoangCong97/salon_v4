## 📌 I. TỔNG QUAN DỰ ÁN (PROJECT OVERVIEW)

Dự án là hệ thống **SaaS Salon App Đa chi nhánh & Đa tenant (SaaS Multi-tenant, Multi-branch Salon App)** cung cấp giải pháp vận hành, đặt lịch hẹn trực tuyến, quản lý nhân viên, chấm công, hóa đơn/thanh toán POS, tính hoa hồng và bảng lương cho các salon.

### Phân vùng 3 Giao diện Gốc (3 Frontends):
1. **Site Nội Bộ (Super Admin Portal)**: Giao diện tối ưu máy tính dành cho Admin tổng để quản lý các Salon (Tenants), cấp giấy phép sử dụng và theo dõi báo cáo doanh thu toàn hệ thống.

   - **Giao diện nhân viên (Mobile Web)**: Dành cho Employee (Thợ chính/phụ) xem lịch hẹn được phân công, thực hiện công việc, chấm công và xem hoa hồng cá nhân hàng ngày.
3. **Site Khách Hàng (Customer Booking Portal)**: Giao diện Mobile-first chạy trên Next.js giúp khách hàng tìm kiếm salon, xem dịch vụ/bảng giá, đặt lịch và đánh giá phản hồi.

## 🔒 IV. QUY TẮC PHÂN TÁCH TENANT (TENANT ISOLATION RULES)

> [!IMPORTANT]
> Đây là dự án SaaS đa tenant, tính bảo mật dữ liệu giữa các salon là ưu tiên tối thượng. AI cần tuân thủ nghiêm ngặt các quy tắc sau:

1. **Xác định Tenant**:
   - Mọi request gửi tới `backend-api` phải được middleware phân giải `tenant_id` từ **Request Header** (`X-Tenant-ID`) hoặc trích xuất từ **subdomain** của URL.
   - Khi đã đăng nhập, `tenant_id` phải được giải mã từ JWT token của người dùng.
2. **Truy vấn Database**:
   - Tất cả các câu lệnh truy vấn dữ liệu (SELECT, UPDATE, DELETE) đối với các bảng có liên kết tenant **bắt buộc phải bao gồm điều kiện lọc `tenant_id`**.
   - **Ví dụ**:
     ```typescript
     // SAI ❌: Truy vấn không kiểm soát tenant
     prisma.service.findMany({ where: { branchId } });

     // ĐÚNG ✅: Luôn luôn có tenantId
     prisma.service.findMany({ where: { tenantId, branchId } });
     ```
3. **Soft Delete (Xóa tạm)**:
   - Tất cả các bảng dữ liệu đều sử dụng cơ chế soft delete (cột `deleted_at`).
   - Mọi câu lệnh SELECT thông thường phải đi kèm điều kiện `deletedAt: null`.

## 💻 V. QUY TẮC LẬP TRÌNH DÀNH CHO AI (AI CODING RULES)

### 1. Database & Prisma
- Prisma Schema tập trung nằm tại [packages/database/prisma/schema.prisma](file:///c:/Workspace/Web09_salon-app/packages/database/prisma/schema.prisma).
- **Tuyệt đối không** import `@prisma/client` trực tiếp trong code ứng dụng khác rồi tạo instance mới. Hãy import và sử dụng instance `prisma` xuất khẩu từ thư viện `@salon/database`.
- **Ví dụ**:
  ```typescript
  import { prisma } from '@salon/database';
  ```

### 2. Định nghĩa Dữ liệu & Types
- Khi viết các tham số API Request, API Response, DTOs hoặc Interfaces, hãy đặt chúng trong `@salon/shared-types` ([packages/shared-types](file:///c:/Workspace/Web09_salon-app/packages/shared-types)).
- Điều này đảm bảo khi Backend thay đổi trường dữ liệu, Frontend sẽ lập tức báo lỗi TypeScript compile, tránh việc API không khớp.

### 3. Tiện ích & Định dạng (Utils)
- Không viết lại các hàm định dạng ngày tháng hoặc tiền tệ ở từng ứng dụng.
- Sử dụng hàm `formatCurrencyVND` và `formatDateVN` có sẵn tại `@salon/shared-utils` ([packages/shared-utils](file:///c:/Workspace/Web09_salon-app/packages/shared-utils)).

### 4. Cấu trúc Module của NestJS (Backend)
- Mỗi tính năng nghiệp vụ lớn cần có Module riêng chứa:
  - `*.module.ts` (Khai báo module)
  - `*.controller.ts` (Routing, xử lý HTTP request)
  - `*.service.ts` (Logic nghiệp vụ và tương tác database)
  - `dto/*.dto.ts` (Định nghĩa dữ liệu đầu vào)
- Không viết logic nghiệp vụ trực tiếp trong Controller. Controller chỉ làm nhiệm vụ nhận, validates đầu vào và trả về output.
- **Chuẩn hóa dữ liệu đầu ra (API Response)**: Tất cả các endpoint của Controller bắt buộc phải trả về kiểu cấu trúc `ApiResponse<T>` từ `@salon/shared-types` để Frontend đồng bộ xử lý.
- **Xác thực và Phân quyền (Auth & RBAC)**:
  - Sử dụng decorator `@CurrentUser()` (đã khai báo trong `common/decorators`) để lấy thông tin session của người dùng đang đăng nhập (`UserSession`).
  - Sử dụng decorator `@RequirePermissions('permission.slug')` kết hợp với `PermissionsGuard` để kiểm tra phân quyền người dùng trước khi vào Controller.
- **Truyền Ngữ cảnh Tenant (Tenant Context Propagation)**:
  - Controller có trách nhiệm lấy `tenantId` và `branchId` từ `UserSession` (lấy qua `@CurrentUser()`) và truyền trực tiếp vào các đối số của Service.
  - **Ví dụ**:
    ```typescript
    @Get()
    @RequirePermissions('booking.view')
    async getBookings(@CurrentUser() session: UserSession) {
      const bookings = await this.bookingService.findAllByTenant(session.tenantId);
      return { success: true, data: bookings };
    }
    ```

## 📊 VIII. BẢN THIẾT KẾ & QUY CHUẨN GIAO DIỆN BẢNG EXCEL COMPACT (EXCEL-LIKE COMPACT TABLE GUIDELINES)

Quy chuẩn này định nghĩa cấu trúc giao diện nhập liệu dạng bảng Excel compact trực quan dành cho các danh sách quản trị (Dịch vụ, Kho hàng, Nhân viên,...) giúp thao tác nhanh và tối ưu hóa diện tích hiển thị.

### 1. Kích thước và Phân cột (Grid & Layout Columns)
* **Độ cao hàng (Row Height)**: Cố định ở `38px` đối với toàn bộ các ô để đảm bảo tính đồng đều.
* **Kích thước chữ (Font Size)**: Thiết lập mặc định `13px` cho gọn gàng (Excel style).
* **Cột Số liệu**: Tất cả các cột số lượng, giá bán, thời lượng, hoa hồng phải được căn giữa (`textAlign: "center"`) cả ở Header lẫn Ô dữ liệu.

### 2. Thiết kế Ô nhập liệu (Excel Input Cell CSS/Styles)
* Thẻ `td` cha phải được cấu hình: `{ padding: 0, verticalAlign: "middle", height: "38px", position: "relative" }`
* Ô nhập liệu `<input>` bên trong sử dụng class `.excel-input`:
  * Chiếm toàn bộ diện tích: `{ width: "100%", height: "100%", border: "none", borderRadius: 0, boxSizing: "border-box" }`
  * Nền trong suốt khi hiển thị bình thường.
  * Khi hover: Chuyển màu nền nhẹ (`hsl(210, 40%, 96%)`).
  * Khi focus (đang sửa): Chuyển màu nền trắng (`white`), outline xanh đậm `{ outline: "2px solid var(--color-primary)", outlineOffset: "-2px" }` và đặt `z-index: 10`, `position: "relative"` để đường outline đè khít lên viền của bảng mà không làm xô lệch bố cục ô.

### 3. Căn phải đơn vị tuyệt đối (Absolute Unit Overlays)
Để tránh đơn vị (phút, đ, %) đè lên số đang nhập và giữ số căn giữa:
* Đơn vị được đặt trong thẻ absolute nằm ở góc phải của `td` cha có relative:
  ```tsx
  <span style={{
    position: "absolute",
    right: "10px",
    top: "50%",
    transform: "translateY(-50%)",
    fontSize: "11px",
    color: "var(--text-muted)",
    pointerEvents: "none"
  }}>
    đơn_vị
  </span>
  ```
* Bổ sung lề đệm bên phải (`paddingRight`) cho `<input>` để văn bản không bị tràn đè lên nhãn đơn vị.
* **Logic thông minh cho giá khuyến mãi**: Chỉ hiển thị ký tự đơn vị tiền tệ `"đ"` khi trường giá trị thực sự tồn tại giá khuyến mãi (khác null và nhỏ hơn giá gốc). Nếu không có giá khuyến mãi, hiển thị placeholder `"--"` và ẩn ký hiệu đơn vị đi.

### 4. Định dạng Dropdown phân loại bo góc (Excel Dropdown Badge)
* Thẻ `td` chứa dropdown có padding lề `{ padding: "3px 6px", height: "38px" }` để chừa khoảng trống xung quanh.
* Dropdown `<select className="excel-select">` có:
  * Chiều cao `32px`, chiều rộng `100%`.
  * Bo góc `{ borderRadius: "6px" }` tạo hình dạng giống một chiếc badge (thẻ phân loại) bo góc mềm mại nằm lơ lửng giữa hàng thay vì khối chữ nhật full ô sắc nhọn.
  * Tận dụng màu sắc theo phân loại làm nền và chữ (`getColorStyle`).
  * Khi focus: Chuyển nền trắng (`white`), chữ tối (`var(--text-primary)`) và outline xanh `{ outline: "2px solid var(--color-primary)", outlineOffset: "-1px" }` để dễ chọn lựa.

### 5. Hành vi Chọn chữ & Định vị con trỏ (First-Click Select All & Double Click)
Người dùng mong muốn click hoặc tab vào ô là bôi đen ngay để gõ đè, nhưng nếu nhấp tiếp/nhấn đúp thì phải hiện con trỏ văn bản thông thường để sửa ký tự:
* Gán sự kiện bôi đen khi focus: `onFocus={(e) => e.target.select()}`
* Chặn sự kiện hủy bôi đen của browser ở lần click đầu tiên bằng `onMouseDown`:
  ```typescript
  const handleMouseDownSelectAll = (e: React.MouseEvent<HTMLInputElement>) => {
    if (document.activeElement !== e.currentTarget) {
      e.currentTarget.focus();
      e.preventDefault();
    }
  };
  ```

### 6. Đồng bộ lưu dữ liệu và Làm mới ngầm (Auto-save & Silent Refresh)
* Hệ thống tự động kích hoạt sự kiện lưu khi mất focus (`onBlur`). Bấm `Enter` sẽ gọi `e.currentTarget.blur()` để đồng bộ logic.
* **Mở rộng hàm tải dữ liệu (Silent Refresh)**: Các hàm nạp dữ liệu như `fetchServices` và `fetchCategories` phải nhận đối số `silent?: boolean`.
  * Khi tự động lưu, gọi `fetchServices(true)` để cập nhật dữ liệu chạy ngầm mà không đặt `loading: true`.
  * Việc này giữ cho bảng React không bị unmount/hiển thị màn hình spinner xoay tròn, duy trì nguyên trạng thái focus/scroll/selection của người dùng khi di chuyển chỉnh sửa liên tiếp qua các dòng khác nhau.
